## 关于这本书

《代码整洁之道》（Clean Code）是 Robert C. Martin（Bob 大叔）的经典著作。书中通过大量实例，阐述了编写整洁代码的原则和实践。

> 写代码就像写文章，首先要让人能读懂。

## 核心要点

### 一、有意义的命名

好的命名应该 **名副其实**：

```java
// 糟糕的命名
int d; // 经过的天数

// 好的命名
int elapsedTimeInDays;
int daysSinceCreation;
```

**命名原则：**
- 使用读得出来的名字
- 使用可搜索的名字
- 避免使用编码前缀（如匈牙利命名法）
- 类名用名词，方法名用动词

### 二、函数

**函数应该做一件事，做好这件事，只做这一件事。**

```python
# 糟糕：一个函数做了太多事
def process_user_data(user):
    # 验证数据
    # 保存到数据库
    # 发送邮件
    # 记录日志
    pass

# 好的：职责分离
def validate_user(user): ...
def save_user(user): ...
def send_welcome_email(user): ...
```

**函数原则：**
- 短小精悍（尽量控制在 20 行以内）
- 参数越少越好（最好不要超过 3 个）
- 无副作用
- 使用异常代替返回错误码

### 三、注释

> 注释的恰当用法是弥补我们在用代码表达意图时遭遇的失败。

**好注释：**
- 法律信息（版权声明）
- 对意图的解释（为什么这样做）
- TODO 标记

**坏注释：**
- 冗余注释（重复代码已经表达的内容）
- 误导性注释
- 日志式注释（应该用 Git 管理）
- 被注释掉的代码（直接删除）

### 四、错误处理

```java
// 糟糕：返回错误码
if (deletePage(page) == E_OK) {
    if (registry.deleteReference(page.name) == E_OK) {
        logger.log("page deleted");
    }
}

// 好的：使用异常
try {
    deletePage(page);
    registry.deleteReference(page.name);
    logger.log("page deleted");
} catch (Exception e) {
    logger.log(e.getMessage());
}
```

## 个人感悟

### 1. 代码是写给人看的

这条原则看似简单，但真正做到需要持续的自律。每次写完代码，问自己：

- 同事能在 5 分钟内理解这段代码吗？
- 三个月后的自己能看懂吗？

### 2. 童子军军规

> 让营地比你来时更干净。

每次修改代码时，让代码质量比之前好一点点。积少成多，代码库就会越来越好。

### 3. 重构的勇气

不要害怕重构。有测试用例的保护，大胆地改善代码结构。技术债务就像贷款，迟早要还。

## 实践清单

- [ ] 变量名能准确表达含义
- [ ] 每个函数只做一件事
- [ ] 删除冗余注释
- [ ] 使用异常处理错误
- [ ] 保持代码格式一致
- [ ] 提交前自查代码质量
